## 一、新增功能概述

本项目在原「健康生活」应用基础上新增了两项功能：

| 功能 | 定位 | 核心价值 |
|------|------|----------|
| **功能一：每日健康知识推送** | 首页信息卡片 | 增加用户停留动力，知识科普引导习惯养成 |
| **功能二：打卡统计图表** | 数据可视化 | 量化打卡数据，帮助用户直观了解习惯趋势 |

两项功能均遵循最小侵入原则，不破坏任何原有逻辑，分别新增/修改 3 个和 6 个文件。

## 二、功能一：每日健康知识推送

### 2.1 功能定位

在首页新增「每日健康知识推送」卡片，用户每次打开应用都可以看到一条有科学依据的健康冷知识。支持展开/收起查看详情，不打断原有打卡流程。

### 2.2 解决的问题

| 问题 | 改进 |
|------|------|
| 原应用仅支持「任务打卡」，用户打开后缺乏停留动力 | 新增知识卡片增加信息获取乐趣 |
| 首页顶部进度区利用率低 | 在进度区和任务列表之间加入知识卡片，丰富页面层次 |
| 健康习惯养成缺乏知识引导 | 30条科学健康知识帮助用户理解"为什么要做这些任务" |

### 2.3 使用场景

- 用户每天早上打开应用打卡时，随机看到一条健康冷知识
- 看到感兴趣的标题可以点击展开阅读全文
- 读完可收起，不影响查看下方的任务列表

## 四、功能一 核心代码解析

### 4.1 每日知识获取算法（KnowledgeData.ets）

```typescript
export function getDailyKnowledge(date?: Date): KnowledgeItem {
  const targetDate = date || new Date();
  // 将年月日组合成一个整数值作为种子
  const seed = targetDate.getFullYear() * 10000
             + (targetDate.getMonth() + 1) * 100
             + targetDate.getDate();
  // Knuth 乘法哈希 → 取模得到数组索引
  const index = (seed * 2654435761) % KNOWLEDGE_LIST.length >>> 0;
  return KNOWLEDGE_LIST[index];
}
```

**设计要点**：
- 使用日期作为随机种子，保证**同一天返回同一条**，不会每次打开都变
- Knuth 乘法哈希（黄金比例 2654435761）保证相邻日期得到离散分布的知识条目
- 支持传入 Date 参数，方便测试

### 4.2 卡片展开/收起动画（KnowledgeCard.ets）

```typescript
@State isExpanded: boolean = false;

.onClick(() => {
  this.isExpanded = !this.isExpanded;  // 切换状态，触发 UI 自动重渲染
})
```

**设计要点**：
- ArkTS 声明式 UI：`isExpanded` 变化后，框架自动增删 `if (this.isExpanded)` 分支内的 DOM 节点
- 收起时标题限制单行 + 省略号（`maxLines(1) + TextOverflow.Ellipsis`），展开时允许最多 10 行
- 整个卡片区域可点击，交互热区大

### 4.3 集成方式（HomeComponent.ets）

**设计要点**：
- 最小侵入性原则：只在原有 `Column` 中新增 1 行组件调用，零风险
- 使用 `@State` 而非 `@Link`：知识数据只读，不需要父子双向同步
- 数据通过 `@Prop` 传入子组件，保持单向数据流

## 五、功能二：打卡统计图表

### 5.1 功能定位

新增第 4 个底部 Tab「统计」，提供打卡数据的可视化分析，包括 Canvas 柱状图展示近 7 天完成率趋势和 4 项汇总统计指标（累计打卡天数、当前连续天数、历史最佳、已获成就数）。

### 5.2 解决的问题

| 问题 | 改进 |
|------|------|
| 原应用只有打卡功能，用户无法了解自己的长期习惯趋势 | 柱状图直观展示每周打卡波动 |
| 缺少数据反馈激励 | 连续打卡天数等指标提供量化成就反馈 |
| 3 个 Tab 内容较简单 | 新增统计 Tab 丰富了应用的信息密度 |

### 5.3 使用场景

- 用户打卡一段时间后想了解自己的习惯养成情况
- 通过柱状图发现哪些天打卡率较低，针对性调整
- 连续天数指标激励用户保持打卡习惯

### 5.4 新增 / 修改文件清单

| 文件路径 | 变更类型 | 说明 |
|----------|:--:|------|
| `viewmodel/StatsViewModel.ets` | **新增** | 数据逻辑层：查询 DayInfo/GlobalInfo，计算连续天数、累计打卡等统计指标 |
| `view/ChartComponent.ets` | **新增** | Canvas 柱状图组件：7 天完成率趋势图，今日蓝色高亮 |
| `pages/StatsPage.ets` | **新增** | 统计页面：Loading 状态 + 4 指标卡片 + 图表 + 每日详情列表 |
| `model/NavItemModel.ets` | **修改** | TabId 枚举新增 STATS，NavList 数组新增统计导航项（第 3 位） |
| `pages/MainPage.ets` | **修改** | 导入 StatsIndex，在 Achievement 和 Mine 之间插入第 4 个 TabContent |
| `resources/base/element/string.json` | **修改** | 新增 tab_stats 字符串资源 |

### 5.5 核心代码解析

**StatsViewModel 数据计算（StatsViewModel.ets）**：

- `queryWeekStats()`：生成近 7 天日期列表，批量查询 DayInfo + GlobalInfo，返回组装好的 `StatsData`
- `calculateCurrentStreak()`：从今天往前遍历 DayStats 数组，百分百完成才累计连续天数，今天未完成不影响（继续往前数）
- `calculateBestStreak()`：解析 GlobalInfo 的 achievements 字段（如 "3,7,30"），取最大值作为历史最佳

**Canvas 柱状图渲染（ChartComponent.ets）**：

- 使用 ArkTS 标准画布模式：`RenderingContextSettings` + `CanvasRenderingContext2D`
- `drawChart()`：绘制参考线（0%/50%/100%）→ 遍历 DayStats 画 7 根柱子 → 柱顶百分比标签 → 柱底星期标签
- 今日柱子用 `#2196F3` 蓝色高亮，底部加圆点标记
- 所有变量显式类型注解，使用 `RefLine` 接口替代裸对象

**统计页面结构（StatsPage.ets）**：

- `aboutToAppear()` 和 `onPageShow()` 均触发 `loadData()`
- `@State statsData` 驱动 UI，加载完成后展示 4 个统计卡片 + 柱状图 + 每日详情行
- 每日详情行包含：日期、进度条（Stack 叠加）、百分比、状态图标（✅/🔄/⬜）
