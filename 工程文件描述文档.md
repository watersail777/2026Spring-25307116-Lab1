## 一、项目整体介绍

### 1.1 功能定位

「健康生活」是一款基于 **HarmonyOS NEXT** 平台开发的健康习惯养成应用。通过任务打卡、成就激励、后台提醒和服务卡片等机制，帮助用户建立并坚持健康生活习惯。

### 1.2 核心业务场景

| 场景 | 描述 |
|------|------|
| **任务管理** | 用户可创建最多 7 项健康任务（早起、喝水、吃苹果、每日微笑、刷牙、早睡、跑步），设置目标量、提醒时间和每周频率 |
| **每日打卡** | 一次性任务（早起/微笑/刷牙/早睡）打卡一次即完成；累计型任务（喝水/吃苹果/跑步）需多次打卡达到目标量 |
| **进度追踪** | 主页面以百分比显示当天任务完成进度，周历视图展示每日完成状态 |
| **成就系统** | 连续打卡达到 3/7/30/50/73/99 天时触发成就动画弹窗，可到成就页查看勋章 |
| **后台提醒** | 通过后台代理提醒向用户发送任务通知（倒计时/日历/闹钟三种类型） |
| **服务卡片** | 2×2 进度卡片（显示今日完成度）+ 2×4 任务卡片（显示任务列表），支持桌面快捷访问 |

### 1.3 项目背景

本项目是 2024 年暑期华为 HarmonyOS 菁英班的应用开发实践作品，基于华为官方 Codelab [HarmonyOS_Codelabs/Healthy_life](https://gitee.com/harmonyos_codelabs/Healthy_life) 修复 bug 并扩展功能。项目在 Gitee 上协作完成，GitHub 为副本镜像。

## 三、核心文件 / 入口文件解析

### 3.1 应用入口：`EntryAbility.ets`

**文件路径**：`entry/src/main/ets/entryability/EntryAbility.ets`

**功能**：作为 Stage 模型的生命周期入口，负责应用的启动初始化。

**关键逻辑**（逐段说明）：

```
onCreate() → 全局上下文初始化 → 创建数据库 → 创建 5 张表 → 插入默认用户信息
onWindowStageCreate() → 加载启动页 SplashPage
onForeground() → 恢复前台状态
onBackground() → 更新服务卡片
```

| 生命周期 | 触发时机 | 核心操作 |
|----------|----------|----------|
| `onCreate` | 应用首次创建 | 初始化 GlobalContext、创建 taskInfo.db 数据库、按序创建 5 张表（dayInfo / globalInfo / taskInfo / formInfo / SelfInfo）、写入默认用户信息 |
| `onWindowStageCreate` | 窗口创建时 | 调用 `windowStage.loadContent('pages/SplashPage')` 加载启动页 |
| `onForeground` | 从后台切回前台 | 设置前台标记为 true，重置任务列表变更标记 |
| `onBackground` | 切到后台 | 调用 `FormUtils.backgroundUpdateCard()` 刷新桌面服务卡片 |
| `onNewWant` | 收到新 Want（如卡片点击） | 更新 want 和 launchParam 到全局上下文 |

**数据库表关系**：

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `dayInfo` | 每日打卡记录 | date, targetTaskNum, finTaskNum |
| `taskInfo` | 任务详细信息 | date, taskID, targetValue, isDone, isOpen, frequency |
| `globalInfo` | 全局状态 | firstDate, lastDate, checkInDays, achievements |
| `formInfo` | 服务卡片信息 | formId, formName, formDimension |
| `SelfInfo` | 用户个人信息 | name, img, birth, gender, hobby, sig |

### 3.2 主页面容器：`MainPage.ets`

**文件路径**：`entry/src/main/ets/pages/MainPage.ets`

**功能**：整个应用的 Tab 导航容器，管理 3 个标签页的切换。

**关键逻辑**：

| 代码段 | 说明 |
|--------|------|
| `@State currentPage: number = 0` | 当前选中的 Tab 索引，变化时触发 Tab 切换动画 |
| `@State homeStore: HomeStore` | 核心状态对象，传入 HomeIndex 子组件并在 onPageShow 时初始化 |
| `aboutToAppear()` | 请求通知权限 |
| `onPageShow()` | 每次页面显示时：① 更新用户昵称/头像 ② 检查是否跨天（跨天则重建 HomeStore） ③ 调用 `checkCurrentTime()` 校验系统时间 |
| `checkCurrentTime()` | 比对 lastDate 与当前时间，若系统时间被回拨则弹窗终止应用，否则调用 `homeStore.initData()` 加载数据 |
| `build()` | `Tabs` 容器 + 3 个 `TabContent`：首页（HomeIndex）、成就（AchievementIndex）、我的（MineIndex） |
| `TabBuilder()` | 自定义底部 Tab 图标和文字样式 |

**3 个 Tab 的数据流**：

```
MainPage
├── Tab 1: HomeIndex   ← @Link homeStore（双向绑定）
│   ├── HomeTopView    ← @Link homeStore（进度 + 周历）
│   ├── TaskCard[]     ← ForEach(homeStore.getTaskListOfDay())
│   └── CustomDialogView ← @Provide broadCast（事件广播）
├── Tab 2: AchievementIndex（成就勋章页）
│   └── BadgePanel     ← AppStorage（从成就系统读取）
└── Tab 3: MineIndex   ← @State nickname/signature（个人资料）
```

**文件路径**：`entry/src/main/ets/view/HomeComponent.ets`

**功能**：首页主体布局，包含滚动任务列表、顶部进度区域和浮动添加按钮。

| 方法 | 说明 |

### 3.4 核心状态管理：`HomeViewModel.ets`

**文件路径**：`entry/src/main/ets/viewmodel/HomeViewModel.ets`

**功能**：用 `@Observed` 装饰的核心状态类，管理首页全部数据。这是整个应用的数据枢纽。

**关键方法**：

| 方法 | 说明 |
|------|------|
| `initData()` | 页面首次显示时调用：① 查询最近 90 天 DayInfo 和 TaskInfo ② 初始化周历数据 ③ 查询当天任务列表 |
| `setSelectedShowDate(showDateTime)` | 点击周历某天时调用 → 从数据库查询该日任务 → 更新 `selectedDayInfo` → 界面刷新 |
| `getDonePercent()` | 计算当天完成百分比 → 驱动顶部进度数字更新 |
| `getTaskListOfDay()` | 返回当前选中日期的任务列表 → ForEach 循环渲染 TaskCard |
| `taskClock(taskInfo)` | **打卡核心**：更新任务完成状态 → 写入数据库 → 更新 dayInfo（finTaskNum+1）→ 若当天全部完成则 checkInDays+1 → 判断是否达成成就 |
| `updateTaskInfoList(editedTaskInfo)` | 编辑任务后回调：更新内存 state → 同步数据库 → 刷新 DayInfo |
| `updateAchievement(dayInfo)` | 连续打卡天数判定：比较前一天是否完成 → 更新 checkInDays → 检查 3/7/30/50/73/99 天成就 |

**数据流转示意**：

```
用户点击打卡
    │
    ▼
taskItemAction() ──→ CustomDialog 弹窗确认
    │
    ▼
onConfirm() ──→ homeStore.taskClock()
    │
    ├─→ updateTask()       写入 taskInfo 表（isDone=true, finValue 更新）
    ├─→ updateDayInfo()    写入 dayInfo 表（finTaskNum+1）
    └─→ updateAchievement() 写入 globalInfo 表（checkInDays 累加）
         │
         └── 满足成就条件？
              ├─ 是 → AppStorage 更新 → 广播 SHOW_ACHIEVEMENT_DIALOG → 弹窗
              └─ 否 → 静默完成
```

### 3.6 数据库核心：`DatabaseModel.ets`

**文件路径**：`entry/src/main/ets/model/DatabaseModel.ets`

**功能**：封装数据初始化逻辑，处理首次启动与后续启动的数据差异。

**关键逻辑**：

```
query(date, callback)
    │
    ├─ GlobalInfoApi.query()
    │     │
    │     ├─ globalResult.firstDate 不存在（首次启动）
    │     │   ├─ 写入 globalInfo（含首次日期）
    │     │   ├─ insertGlobalTask()：插入 7 条模板任务（date='global'）
    │     │   ├─ 写入当天 dayInfo
    │     │   └─ insertTask(date)：插入当天 7 条任务实例
    │     │
    │     └─ globalResult.firstDate 存在（非首次启动）
    │         ├─ 更新 lastDate 为今天
    │         └─ queryPreInfo()：查询今天任务
    │               │
    │               ├─ 今天有任务 → calFinishNum() 计算完成数 → 返回
    │               └─ 今天无任务 → 从 global 模板复制 → insertPreInfo() 补填历史缺失日期的数据
```

### 4.2 完整数据流转（用户打卡为例）

```
1. 用户在首页点击任务卡片
   HomeComponent.taskItemAction(item, isClick=true)
        │
2. 通过 BroadCast 广播事件
   broadCast.emit(BroadCastType.SHOW_TASK_DETAIL_DIALOG, [item, callback])
        │
3. CustomDialogView 接收到广播，弹出打卡确认弹窗
   TaskDetailDialog 展示
        │
4. 用户点击确认 → 回调 callback.confirmCallback(task)
   HomeComponent.onConfirm(task)
        │
5. 调用 HomeStore.taskClock(task)
   ├─ updateTask() → TaskInfoApi.updateDataByDate()  → 写入 taskInfo 表
   ├─ updateDayInfo() → DayInfoApi.updateData()  → 写入 dayInfo 表
   └─ updateAchievement() → GlobalInfoApi.updateData()  → 写入 globalInfo 表
        │
6. 返回 AchievementInfo { achievementLevel, showAchievement }
        │
7. showAchievement = true → 广播 SHOW_ACHIEVEMENT_DIALOG
   → AchievementDialog 弹出成就动画
        │
8. @Observed HomeStore 状态变化 → 触发 UI 自动刷新
   ├─ HomeTopView 进度百分比更新
   ├─ TaskCard 状态更新（打勾/进度条）
   └─ WeekCalendar 日期标记更新
```

### 4.3 状态管理机制

| 机制 | 使用场景 | 示例 |
|------|----------|------|
| `@State` | 组件内部状态 | MainPage 的 currentPage、MinePage 的 nickname |
| `@Link` | 父子组件双向绑定 | HomeStore 从 MainPage → HomeIndex → HomeTopView |
| `@Provide / @Consume` | 跨层级注入 | BroadCast 事件总线 |
| `@Observed / @ObjectLink` | 类级别可观察 | HomeStore 类被 `@Observed` 装饰，属性变化自动触发渲染 |
| `@Watch` | 属性变化监听 | MainPage 中 `@Watch('taskChange') editedTaskID` |
| `AppStorage` | 应用级全局状态 | 成就等级 `ACHIEVEMENT_LEVEL_KEY` |
| `BroadCast`（自研） | 事件总线 | 弹窗显示/隐藏事件 |
